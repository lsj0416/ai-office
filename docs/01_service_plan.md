# AI Office — 서비스 기획서

> **"내 맘대로 AI 오피스"** — 역할 기반 멀티 에이전트를 시각적으로 구성하고, 1인 개발자가 실제로 쓰는 AI 팀

---

## 1. 서비스 개요

### 배경 및 문제 정의

1인 개발자 / 소규모 스타트업은 개발 이후 단계에서 막힌다.

- 서비스는 만들었는데 런칭, 마케팅, 운영, 수익화를 혼자 감당해야 함
- 각 영역의 전문가를 고용하기엔 비용이 너무 큼
- 기존 AI 도구(ChatGPT 등)는 우리 회사 맥락을 모름 → 매번 배경 설명 반복

### 핵심 아이디어

> 회사 정보를 입력하면 AI 직원팀이 자동 생성되고,
> 사무실 공간에서 직원들이 실제로 일하듯 움직이며,
> 쓸수록 우리 회사 히스토리를 아는 AI팀이 되는 서비스

### 페르소나

**메인 타겟: 1인 개발자 (= 세종씨 본인)**

```
백엔드 개발자
사이드 프로젝트 여러 개 동시 진행 중
개발 이후 마케팅 / 운영 / 수익화가 막막
툴이 너무 많아서 업무가 파편화됨
전문가 고용 비용이 부담됨
```

**확장 타겟**
- 10인 이하 소규모 스타트업
- 1인 크리에이터 / 유튜버
- 소규모 마케팅 대행사
- 전문가 없는 초기 팀 (디자이너, 마케터 없는 개발팀 등)

## 2. 핵심 기능

### 2-1. 회사 생성

회사 이름 + 비전 + 핵심 비즈니스 입력 → AI가 자동으로 조직 구성

**자동 생성되는 것들**
- 서비스 도메인에 맞는 역할 추천 (PM, 개발자, 마케터, 도메인 전문가 등)
- 각 역할별 페르소나 (회사 맥락이 주입된 시스템 프롬프트)
- 첫 번째 할 일 목록 제안

**예시 — 예비자(교회 예배 준비 서비스) 기준**
| 역할 | 담당 업무 |
|------|----------|
| PM 지민 | 스프린트 관리, 일정 조율 |
| 개발자 태양 | 기술 스펙, 코드 리뷰 |
| 신학자문 요셉 | 성경 본문 검토, 저작권 자문 |
| 마케터 유나 | 교회 커뮤니티 홍보, 런칭 전략 |
| 리뷰어 민준 | QA, 결과물 피드백 |

### 2-2. AI 오피스 공간 (시각적 사무실)

**현재 버전 (HTML 카드 UI)**
- 역할별 책상 카드
- 현재 작업 상태 표시 (말풍선)
- 클릭 → 해당 직원과 채팅

**다음 버전 (Pixi.js 기반 2D 공간)**
- 타일맵 사무실 맵
- 세종씨 캐릭터 WASD 조작
- 직원들 출퇴근 애니메이션
- 회의 시작 → 직원들이 회의실로 이동
- 공간별 다른 대화 톤 (탕비실 = 캐주얼, 발표실 = 프레젠테이션 모드)

**공간 구성**
```
1층  업무공간 (개발/기획/비즈니스 구역)
2층  회의실 / 발표실
3층  탕비실 / 휴게공간
옥상  브레인스토밍 존
지하  서버실 (개발자 전용)
```

### 2-3. 실행 모드 3가지

| 모드 | 설명 | 적합한 상황 |
|------|------|------------|
| **AUTO** | 입력 하나로 여러 역할이 병렬 처리 → 결과 합산 | 빠르게 결과만 원할 때 |
| **PIPELINE** | 역할들이 순서대로 이어받아 점진적 완성 | 단계적 품질 중시 |
| **CHAT** | 사용자가 중간에 개입하며 각 역할과 대화 | 세밀하게 컨트롤 |

### 2-4. 컨텍스트 축적 (쓸수록 똑똑해지는 AI팀)

단순 API 호출이 아닌, 우리 회사 히스토리를 아는 AI팀으로 발전

**축적되는 데이터**
- 사내 메신저 대화
- 문서 (기획서, 보고서 등)
- 회의록 (STT 자동 변환 + 요약)
- 작업 결과물 및 피드백

**RAG 기반 검색 주입**
- 전체 히스토리를 매번 넣지 않음
- 현재 작업과 관련된 기억만 꺼내서 주입
- 비용 90% 절감 + 속도 유지

**계층형 요약**
```
날것 대화 → 일별 요약 → 주별 요약 → 월별 요약
                                        ↑
                                  AI에게 주입
```

### 2-5. 음성 회의

- 미팅룸 입장 → 음성 자동 녹음 (STT: Whisper)
- 역할마다 다른 목소리 (TTS: ElevenLabs)
- 회의 종료 시 자동 요약 + 액션 아이템 추출 + 태스크 등록

## 3. 사용 플로우

### 온보딩 플로우
```
1. 회사 정보 입력 (이름 + 비전 + 비즈니스)
2. AI가 조직 구성 추천
3. 역할 커스텀 or 그대로 수락
4. 사무실 입장 → 직원들과 첫 대화
```

### 일반 업무 플로우
```
사무실 입장
→ 직원 책상 클릭 or 말 걸기
→ 작업 요청 (런칭 전략 짜줘 / 이 코드 리뷰해줘 등)
→ 결과물 받기
→ 피드백 → 개선 반복
```

### 회의 플로우
```
회의 시작 버튼
→ 직원들이 회의실로 이동
→ 안건 입력 or 음성으로 자유 진행
→ AI들이 각자 역할에 맞는 의견 제시
→ 종료 시 회의록 자동 생성
→ 액션 아이템 → 태스크로 자동 등록
```

## 5. MVP 범위

### Phase 1 — 핵심 검증 (1~2주)
- [ ] 회사 정보 입력 → AI 팀 자동 생성
- [ ] 역할별 채팅 (Claude API 실제 연동)
- [ ] 태스크 관리 (생성 / 완료 체크)
- [ ] 활동 로그

- [ ] Pixi.js 기반 2D 사무실
- [ ] 세종씨 캐릭터 WASD 이동
- [ ] 직원 출퇴근 애니메이션
- [ ] 회의실 이동 + 회의 모드
